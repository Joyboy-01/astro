## Background

FourierGPT讲了这样一件事情：我们可以通过序列化的不确定性来启发式的判定一句话是机器生成还是人类写的。FourierGPT 的整体流程分为三个步骤：估算并归一化似然分数 → 傅里叶变换得到频谱 → 在频谱上分类。

### Step 1：估算并归一化似然分数

**输入**：一段待检测的文本，共 $N$ 个 token：$t_1, t_2, \ldots, t_N$ 用一个固定的预训练语言模型（论文实验中主要使用 Mistral-7B）对输入文本做前向传播，计算每个 token 的原始似然分数（即负对数概率）：

$$s_i = -\log P(t_i \mid t_1, \ldots, t_{i-1})$$

这被称为 estimator model 给出的分数，他代表了文字生成的不确定性。

接着对原始分数在序列内做 z-score 归一化，得到归一化似然序列：

$$\tilde{s}_i = \frac{s_i - \mu}{\sigma}, \quad \mu = \frac{\sum s_i}{N}, \quad \sigma = \sqrt{\frac{\sum(s_i - \mu)^2}{N-1}}$$

原始 $s_i$ 的值依赖于 estimator 模型的选择（更大的模型通常给出更小的值），而 z-score 后的 $\tilde{s}_i$ 表征的是序列内部的相对似然水平，对 estimator 的依赖性更小。原始似然分布高度偏斜，z-score 后更接近正态分布，在后续分类步骤中效果更好。

**输出**：归一化的似然序列 $\tilde{s}_0, \tilde{s}_1, \ldots, \tilde{s}_{N-1}$（时域信号）

### Step 2：傅里叶变换

**输入**：z-score 归一化后的似然序列 $\tilde{s}_0, \ldots, \tilde{s}_{N-1}$ 对该序列施加离散傅里叶变换（DFT）：

$$X(\omega_k) \triangleq \sum_{n=0}^{N-1} \tilde{s}_n e^{-j\omega_k n}$$

结果是一组复数 $\mathcal{F} = \{X(\omega_k)\}_{k=0,\ldots,N-1}$，即输入时域信号的频域表示。其中 $X(\omega_k) = \text{Re}(X(\omega_k)) + \text{Im}(X(\omega_k)) \cdot j$，而频谱强度（幅值）为：

$$\|X(\omega_k)\| = \sqrt{\text{Re}(X(\omega_k))^2 + \text{Im}(X(\omega_k))^2}$$

最终使用幅值序列 $\{\|X(\omega_k)\|\}_{k=0,\ldots,N-1}$ 作为似然的频谱视图。

直觉上，$\|X(\omega_k)\|$ 代表似然分数在以 $1/\omega_k$ 个 token 为周期的模式下的强度。频谱视图将时域中复杂的、难以捕捉的变化模式压缩为更紧凑的表示，从而放大了人类文本和模型文本之间的细微差异。

**输出**：频谱幅值序列 $\{\|X(\omega_k)\|\}$


### Step 3：分类

论文提供了两种分类方式：

**3a. 监督学习分类器（Supervised Classifier）**

**输入**：有标签的频谱数据（人类 or 模型）

训练时使用增强频谱作为输入特征。方法是对似然序列做多轮循环移位（circularization）：给定原始序列 $C_0 = s_1, s_2, \ldots, s_n$，第 $T$ 步循环移位将头部长度为 $T$ 的段截断并拼接到尾部，得到 $C_T = s_{T+1}, \ldots, s_n, s_1, \ldots, s_T$。对每个移位后的序列做傅里叶变换，共得到 $n$ 个频谱，最终取平均频谱 $\bar{\mathcal{F}} = \frac{1}{n}\sum \mathcal{F}(C_t)$ 作为分类器的输入特征。其直觉是：若原始信号中存在弱周期性，多个变体的平均可以放大原本难以检测的周期。论文中使用 SVM 等多种分类器。

**输出**：二分类预测（Human / Model）

**3b.Pair-wise Heuristic-based Classifier**

**输入**：一对来自同一 prompt 的文本（一人类、一模型）的频谱

论文发现似然频谱在低频端存在显著差异：在大多数数据集和模型组合中，模型文本的低频功率幅值大于人类文本。据此设计启发式规则：

$$\left| \sum_{k=1}^{\delta_k} \|X_{\text{Human}}(\omega_k)\| - \sum_{k=1}^{\delta_k} \|X_{\text{Model}}(\omega_k)\| \right| > \varepsilon$$

其中 $\delta_k$ 是一个整数阈值，决定选取多少个低频成分进行比较；$\varepsilon$ 是区分两者幅值差异的实数阈值（实验中取 $\varepsilon = 0$）。若模型文本在低频端的幅值之和大于人类文本，则判定模型文本为机器生成。

**输出**：该对中哪一个是人类写的，哪一个是模型生成的。

## Thinking

当时读完这篇论文我主要想的就是这个启发式发现的结论是什么意思，这个低频是什么意思。当时我记得ai已经可以回答这类问题了，然后我记得当时问出来的结论就是文章在长距离相关性上可以得到更大的差距，现在的视角其实就是NLL序列变化的快慢。

### Example

低频的波，周期很长，意味着要隔很多个token才能看到一次完整的起伏。在NLL序列里，这对应的是：**文章在大范围、长距离上的整体趋**比如：

- 一篇文章前半段整体NLL都偏高（话题生僻），后半段整体偏低（收尾常见句式）
- 整篇文章的"风格是否一致"、"主题有没有漂移"——这也是低频能捕捉的

高频的波，周期很短，意味着相邻几个token之间NLL就发生了剧烈起伏。在NLL序列里，这对应的是：**局部的、突发的不确定性跳变**。比如：

- 一个罕见词突然出现，NLL从低跳到高，下一个词又回落——这就是高频变化
- 上下两句话语义跳跃、逻辑不连贯——在NLL上也会表现为局部的高频波动

```text
高频差异（局部跳变）→ 容易被改写抹平
  改个词、换个句式 → 高频特征就变了

低频差异（全局节律）→ 很难被改写抹平
  即使换了词，整篇文章的生成节律没变 → 低频特征依然在
```

| | 低频 | 高频 |
|---|---|---|
| **NLL序列里的含义** | 长距离、全局的概率趋势变化 | 相邻token间的局部突变 |
| **文本意义** | 整篇风格一致性、主题稳定性 | 局部语义跳跃、罕见词出现 |
| **机器文本特征** | 低频功率大、全局更平滑规律 | 高频功率小、局部很少突变 |
| **人类文本特征** | 低频功率相对分散 | 高频功率更大、局部更不规则 |
| **判别力** | **强**（稳定，难被改写抹平） | 弱（容易被换词等攻击破坏） |
| **FFT的问题** | 把全文低频混在一起，局部被抵消 | — |
| **小波的改进** | 能定位低频能量来自哪一段 | 细节分辨率好但差异本来就小 |

## 当时的思路

当时的[期末总结](/FourierGPT/从Fourier到Wavelet.pdf)可以回忆起我当时的想法。当时我想的是：对于一整段，我能不能发现到底哪一个部分是机器生成的，或者哪一部分最有可能是机器生成的。当时正好看到了小波变换的介绍视频，于是就想着试一下小波变换能不能work。然后就把傅里叶GPT第三步（DFT）换成了小波变换，并且测试了两种变换的方式（CWT和DWT）。（可惜的是当时虽然提取到了位置信息但是并没有）

因为傅里叶变换最显著的缺陷就是只能捕捉到一个维度的信息，而忽略了位置信息。在当时总结为：FourierGPT的DFT给出的是全局频谱，但无法区分低频能量来自文本的哪一段。比如人类文本里某一段有强低频信号，被另一段的平稳部分抵消，最终全局低频功率反而跟机器文本差不多——这是FourierGPT会误判的情形。引入小波变换，就是为了加上"时间/位置"这个维度。这在之后的工作被总结为易于被攻击：只改了某个部分的词，让那两处概率变低，这样傅里叶变换的全局平均被稀释了，几乎感知不到这个局部变化。

### CWT：连续小波变换

输入是NLL序列，长度N：`[NLL(0), NLL(1), ..., NLL(N-1)]`，CWT用一个母小波 $\psi$，通过改变两个参数来扫描这个序列：

- **$b$（平移）**：把小波窗口从左到右滑动，$b=0$ 对准第0个token，$b=1$ 对准第1个token，以此类推
- **$a$（尺度）**：把小波压缩或拉伸，$a$ 小则窗口窄（看局部高频），$a$ 大则窗口宽（看全局低频）

对每一个 $(a, b)$ 组合，做一次内积：

$$W(a, b) = \frac{1}{\sqrt{a}} \sum_{t=0}^{N-1} \text{NLL}(t) \cdot \psi^*\!\left(\frac{t-b}{a}\right)$$

这个内积的意义是：**在位置 $b$ 附近，用尺度 $a$ 的粒度观察时，NLL序列和这个尺度的波形有多相似**。

输出是一张二维矩阵，行是尺度，列是位置：

```text
        b=0   b=1   b=2  ...  b=N-1
a=1  [  w    w    w   ...   w  ]   ← 小尺度，高频，每个位置的局部波动
a=2  [  w    w    w   ...   w  ]
a=4  [  w    w    w   ...   w  ]
a=8  [  w    w    w   ...   w  ]   ← 大尺度，低频，每个位置的全局趋势
```

取幅值平方就得到能量：

$$\text{Power}(a, b) = |W(a, b)|^2$$

当时讨论了两种方式来利用这个特征：

**方式A（被放弃的）：**

把每一列（每个位置）的所有尺度能量加起来：

$$\text{feature}(b) = \sum_a \text{Power}(a, b)$$

得到一个长度N的向量，代表"文本每个位置的总活跃程度"。问题是这个依赖于机器文本在特定位置有固定模式（比如总是Yes开头），换数据集就失效了。

**方式B（最终采用的）：**

把每一行（每个尺度）的所有位置能量加起来：

$$\text{feature}(a) = \sum_b \text{Power}(a, b)$$

得到一个按尺度排列的向量，代表"这个频率在全文的总体能量"。选定 $k=500$（大约累加到 freq=8 的所有power），作为分类特征。

### DWT

DWT不是在连续参数上扫描，而是对信号做**逐级滤波+下采样**，每级把信号拆成两半：

```text
原始NLL序列（长度N）
        ↓ 低通滤波 + 下采样2
    近似信号（长度N/2）  →  继续分解...
        ↓ 高通滤波 + 下采样2
    细节系数W0（长度N/2）  ← 这一级的高频部分，不再分解
```

level=3时，三级分解的过程是：

```text
NLL（长度N）
├── 高通 → W0（长度N/2）     第1级细节系数，最高频
└── 低通 → 近似1（长度N/2）
    ├── 高通 → W1（长度N/4）  第2级细节系数，中频
    └── 低通 → 近似2（长度N/4）
        ├── 高通 → W2（长度N/8）  第3级细节系数，次低频
        └── 低通 → V0（长度N/8）  最终近似系数，最低频
```

输出：

```python
coeffs = [V0, W0, W1, W2]
```

| 系数 | 长度 | 对应频率范围 | 含义 |
|---|---|---|---|
| $V_0$ | N/8 | 最低频 | 最粗糙的全局趋势 |
| $W_2$ | N/8 | 次低频 | 较慢的变化 |
| $W_1$ | N/4 | 中频 | 中等速度的变化 |
| $W_0$ | N/2 | 最高频 | 相邻token间的快速跳变 |

每个系数数组里的每个元素，都对应原始序列某个位置上的该频率成分——这就是小波比DFT多出来的时间信息。对每个系数数组，把所有元素的平方加起来，得到该频率级别的总能量：

$$\text{feature}(V_0) = \sum_i V_0(i)^2$$

最终特征向量就是：

```text
[Energy(V0), Energy(W2), Energy(W1), Energy(W0)]
  最低频        次低频       中频        高频
```

当时依旧采用了fourierGPT的结论：低频的区分度最大，所以在DWT最低频（freq=0）的power和成为了最有判别力的区分点。但是当时对成功率提高的总结都并不正确。我记得当时也没有想清楚，但是现在看过现在最后发表的论文的思路，我认为应该是这样的：

### DWT部分

DWT在多数数据集上取得了比FFT更好的结果（最高达到1.0000），但成功的原因不是"保留了位置信息"。

真正的原因是：**DWT通过多级低通滤波+下采样，比DFT更干净地提取了低频信号。** DFT是对整个序列一次性做全局变换，序列中任何一个位置的高频噪声都会污染最终的低频结果。DWT则是逐级剥离高频成分，每一级都在过滤掉不需要的部分，最终得到的近似系数$V_0$是经过三次过滤后剩下的纯粹低频，信噪比远高于DFT直接给出的低频系数。

值得注意的是，DWT的$V_0$本来包含粗粒度的位置信息（N个位置被压缩到N/8个点），但我们在最后对$V_0$做了求和：

$$\text{feature} = \sum_i V_0(i)^2$$

这一步把位置信息也消掉了，最终只得到一个标量。所以DWT的实际使用方式和FourierGPT的差距比预期小——两者都是"某个频率段的全局总能量"，只是DWT的低频更纯净。如果不做最后的求和，而是把$V_0$的每个元素单独作为特征，可能能进一步提升效果，但这个方向没有被验证。

### CWT部分

CWT没有取得理想效果，但原因不是"大scale噪声多"或"小scale差异不大"。

真正的原因是：**特征提取方式没有利用CWT的核心优势。** CWT输出的是二维时频能量图，每个格子同时包含位置和频率信息。但我们采用"固定频率，对时间求和"的方式提取特征，把时间维度加总消掉，最终退化成了一个按频率排列的一维向量——和DFT的输出形式完全一样，CWT好不容易保留下来的位置信息在这一步被放弃了。

## 最终的结构

当时思路还有一个显著短板是必须是pairwise的，这个是很有限制的，因为我们在鉴别的时候大多只有一个文本。我认为最终的论文在刚才的发线上加入了一些其他的机制：

**WAVEDETECT：**

```text
条件概率序列 → CWT → 二维谱图 → CNN → 分类
```

**1. 把NLL换成条件概率：**

用原始条件概率 $p_t = P(x_t \mid x_{<t})$ 作为信号。其理论解释是：机器的decoding策略（top-p截断、greedy）相当于对概率信号做了低通滤波，压制了高频成分，使机器文本的概率序列更平滑规律。因此检测机器文本等价于识别这种频谱上的低通特征。

**2. 可学习的surrogate语言模型：**

用一个语言模型（Qwen2.5-0.5B）生成概率序列，但不固定它的参数。先冻结它训练CNN（warm-up），再解冻联合训练。这样语言模型会主动调整自己的输出，使人类和机器文本的概率序列在频谱空间上的差异尽可能大，而不只是被动地输出概率。

**3. 把手工求和换成CNN：**

CWT输出的二维谱图（横轴是token位置，纵轴是尺度/频率）直接送进ResNet-18。CNN自动学习哪些时间-频率位置的组合是最有判别力的模式，而不需要人工决定如何压缩二维信息，从而完整保留并利用了谱图的结构。
